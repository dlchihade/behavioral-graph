Construction

How each part of the graph is built

Four moves take you from raw behavior to an auditable graph. Each is a method, not an assumption.

Nodes · decision states

Latent states become the state layer

Rather than pre-defining "acquisitive" or "attriting," the states are recovered from sequential behavior — the same way exploration and exploitation phases are recovered from a search trajectory. Each unit's state is a distribution, not a hard label.

latent state → decision-state node (estimated, with uncertainty)
Edges · causal + associative

Transitions and influence become edges

State-to-state dynamics give directed edges, and co-occurrence over behavioral sequences gives associative ones. An edge only earns the label "causal" when its effect survives a causal test — holding under conditioning — so it carries an effect size and interval rather than a bare correlation.

transition + co-occurrence → edge · effect survives conditioning → "causal"
Nodes · contextual triggers

External signals enter as triggers

Market shifts, life events, macro trends and a segment's sentiment on current events become contextual-trigger nodes with edges into the states they move. This is the layer refreshed daily — the graph's structure is stable; its activation is current.

external signal + sentiment → contextual-trigger node → state
Property · auditability

Every node and edge carries uncertainty

Because states are estimated and edges are tested, any content suggestion traces back to a specific set of edges with effects and intervals. Nothing the LLM says is free-floating; the grounding is inspectable and reproducible.

estimated node/edge → auditable grounding for LLM content

Latent states turn a static ontology into a measured, moving graph — one you can question edge by edge.
